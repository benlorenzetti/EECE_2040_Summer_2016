Dear Grader: I love plain old C and am not convinced that object oriented programming is worth it. Then again, the whole object[1] of this assignment is learning OOP, so I have decided to use C in an objecty way and then observe what advantages this offered and what advantages C++ would have offered.

C++ got started while Bjarne Stroustrup was working on "C with Classes", so I think this is a very natural way to learn OOP and where its value lies. As an example of some OO techniques I have used, I have parametric polymorphism in a generic container library similar to the std::vector class in C++. [2] Also I use subtype polymorphism with a union of several structs representing different geometric objects. Finally I have ad-hoc polymorphism in an area() function that works on several of the subtypes.

  1. Object, what an overloaded word in computer science, am I right?
  2. Yet another overloaded word: vector; what if I want to store geometric vectors in a dynamic array?
